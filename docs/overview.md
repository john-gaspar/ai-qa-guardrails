# Overview

This template provides guardrails and workflows for QA engineers/SDETs doing API and integration testing with AI assistance. It includes prompts, checklists, evaluation rubrics, workflows, and example evidence to keep outputs safe, repeatable, and reviewable.

Key parts:
- `PROMPTS/`: structured prompts for plans, cases, automation, RCA, and data.
- `CHECKLISTS/`: AI usage, PR, and automation definition of done.
- `EVALS/`: evaluation playbooks, rubrics, contract and flake handling.
- `examples/`: sample end-to-end case with prompts, outputs, edits, and verification.
- `.agent/`: rules and workflows to run projects consistently.
- `docs/`: strategy, stack, devops, risk, quality bar, and evidence guides.
