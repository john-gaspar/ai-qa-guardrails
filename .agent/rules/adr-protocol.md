# ADR Protocol (Automation Decisions)

Use for documenting significant QA/automation decisions.

1) Title + date + owner.
2) Context: requirement, constraints, risks (link `docs/risk-rubric.md`), related issues.
3) Decision: chosen approach and options rejected.
4) Consequences: benefits, trade-offs, monitoring/rollback.
5) Evidence: links to prompts, AI outputs, tests, and `examples/` artifacts.
6) Status: proposed/accepted/deprecated; link follow-up tasks.
